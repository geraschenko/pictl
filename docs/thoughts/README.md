# Thoughts

This directory is for informal observations, half-formed proposals, design doubts, and ideas to revisit later.

Docs here are exploratory, incomplete, and non-authoritative. They are not implementation plans and they do not supersede `docs/design-decisions.md` or `docs/implementation-plan.md`. The goal is to preserve context and uncertainty so future design sessions have a useful starting point instead of rediscovering the same questions.

Good thought docs should usually capture:

- the observation or pain point;
- why it might matter;
- possible directions to explore;
- open questions and uncertainty;
- enough concrete examples to restart the discussion later.

Avoid writing these as settled recommendations unless a decision has actually been made elsewhere.
